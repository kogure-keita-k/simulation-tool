# CHANGELOG ― DPS 試算アプリ

バージョンは改修規模で自動更新（大＝新画面/算出方法の追加削除、中＝算出方法・実装済み機能の修正、小＝見た目）。
※ 出力資料(PPTX等)関連は「機能追加」以外はすべて軽微（パッチ）扱い。
※ 削減・回収の中核式（calc_simulation）は全期間を通じて**無改変**。

---

## v1.11.4（バグ修正：数量が空だと「cannot convert float NaN to integer」）
- ハードウェア関連費・見積機材などの金額集計で、テーブルの**空行（数量/単価がNaN）**があると `ValueError: cannot convert float NaN to integer` で落ちる不具合を修正。
  - 原因：`int(r.get("数量", 0) or 0)` は NaN が真値のため NaN を返し、`int(NaN)` で例外化していた。
  - 対処：NaN/None/空/非数を安全に0へ丸める共通ヘルパー `_num0()` を追加し、数量・単価の集計/表示（HW費合計・概算HW費・見積機材・室外機1台あたり数量・温湿度センサー数・換算表記）を全て置換。
- 中核式（calc_simulation）は無改変。

## v1.11.3（本業商品設定の入力項目を撤去）
- 本業売上換算スライドの削除に伴い、シミュレーション画面の投資設定内「本業商品設定」（主力商品単価・粗利益率）の**入力項目を撤去**。
- 値は各種設定のマスタ（item_price / item_margin）をそのまま使用するため、Excel等に残る本業換算表記は従来どおり動作（各種設定での編集は引き続き可能）。

## v1.11.2（バグ修正：インポート後にテーブル行を削除するとエラー）
- 機体入力・ハードウェア関連・見積の機材テーブルで、**行を手動削除するとエラー**（特にインポート後）になる不具合を修正。
  - 原因：これらの `data_editor` に固定 key を付けたまま、編集結果を毎回 `session_state` へ書き戻していたため、Streamlit が保持する「削除/追加」の編集状態が新しいデータに**二重適用**され、削除行のインデックスがずれて例外化していた。
  - 対処：3つのエディタの key を外し、**入力データ駆動**（毎回の入力DataFrameをそのまま表示）に変更。削除・追加・インポート差し替えが正しく反映される。
- 中核式（calc_simulation）は無改変。

## v1.11.1（バグ修正：機体入力エディタの型不整合）
- 機体入力テーブルで「機器ID/場所」等の列が空だと pandas が float 型と推定し、`StreamlitAPIException: column type text ... not compatible ... FLOAT` が発生していた不具合を修正。エディタ生成前に文字列列を object 型へ明示キャスト（稼働係数は数値、制御可否は○/×へ正規化）。インポート／スナップショット復元後に空列になっても再発しないよう恒久対処。
- 中核式（calc_simulation）は無改変。

## v1.11.0（データ管理・スナップショット受け渡し・案件比較）
- **各種設定に「データ管理（バックアップ）」タブを追加**：
  - `dps_app_data.json` を **丸ごとバックアップDL**（保存案件/スナップショット/企業データ件数を表示）。
  - バックアップ .json を **アップロードして完全復元**（確認チェック付き・上書き注意表示）。
- **スナップショット単体の受け渡し**：入力条件スナップショットを **1件だけ .json で書き出し／取り込み**。JSON全体を共有せず、案件の入力条件だけを別PC・他担当へ渡せる（同名は上書き）。
- **結果一覧に「並べて比較」を追加**：保存案件を2〜3件選び、業態・空調台数・初期費用・年間削減量・年間削減額・回収年数を横並び比較。
  - 比較で初期費用を出すため、保存案件レコードに `invest`（初期費用）を追加保存（旧案件は「—（旧保存）」表示）。
- 中核式（calc_simulation）は無改変。

## v1.10.0（出力ファイル名規則・結果一覧の検索・入力スナップショット）
- **エクスポート ファイル名を統一**：`エクスポート年月日_企業名_種別.拡張子`（例 `20260710_ゼンケン_提案書.pptx`）。敬称（御中/様/株式会社）・空白・使用不可記号を自動除去。提案書PPTX／試算Excel／削減レポートPDF・Excel／請求書PDF・Excel／結果画面のPNG（グラフ・制御可否リスト・サマリカード）に適用。
- **結果一覧に検索を追加**：企業名・業態でしぼり込み（該当件数表示）。削除は検索結果から選んでも元データの正しい位置に反映。
- **入力条件のスナップショット保存＆呼び出し**：シミュレーション画面に「💾 入力条件のスナップショット」を追加。顧客名・業態・検針票12ヶ月・台数（概算/手動）・概算前提・初期費用設定・分析オプションを1件として保存し、後日・別PCで呼び出して同じ条件を再現（同名は上書き／削除可）。データは `dps_app_data.json` の `input_snapshots` に保存。
  - 復元を確実にするため、顧客名・業態・手動台数/材料費/工事費/エリアの各入力欄をキー付きウィジェットに統一（挙動は同じ）。
- 中核式（calc_simulation）は無改変。

## v1.9.0（概算の前提を案件ごとに手動調整）
- 概算モードON時に **「🔧 概算の前提（この案件用に調整）」** パネルを追加。従来は全自動だった前提値を、案件単位で手入力・上書き可能に（既定はマスタ値を反映）。
  - **標準室外機の定格出力（kW/台）**：概算台数＝空調ピーク負荷÷この定格。定格を変えると推定台数が即時に変わる。
  - **概算 制御比率（％）**：制御台数の初期値。
  - **空調ピーク比率の上書き（％／0＝業態マスタ）**。
  - **材料費・工事費（円/台）／ハードウェア固定費・単価（円）**：初期費用の各概算メトリクスへ即反映。
- `estimate_units_from_meter()` に `overrides` 引数を追加（後方互換：未指定はマスタ既定）。台数手入力（総台数・制御台数）は従来どおり。
- 中核式（calc_simulation）は無改変。

## v1.8.0（制御可否リストの列選択・画像出力のExcel取込）
- **制御可否リスト画像に列選択を追加**：系統名／設置場所／メーカー／型式／制御可否から不要な列を外して生成可（シミュレーション画面の「スライド貼付用 画像出力」＋各種設定「画像出力」の両方）。列を外すと残り列の幅を自動再配分。
- **各種設定「画像出力」②の制御可否を〇×表記に変更**（従来のチェックボックス→〇/×セレクト。シミュレーション画面と統一）。
- **各種設定「画像出力」にExcel取込を追加**：試算Excel（『入力』シート→①グラフ／『室外機情報』シート→②制御可否リスト）を取り込んで各表へ自動反映。横並びグリッド／フラット表のどちらのレイアウトも自動判定。※Excelは形式の一例で値は焼き込まず、アップロード内容のみ反映。
- 中核式（calc_simulation）は無改変。
