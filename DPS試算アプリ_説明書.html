DPS 空調デマンド制御
提案・稟議シミュレーター 説明書

検針票12ヶ月のデータから、空調デマンド制御による電気料金の削減効果と投資回収を試算し、提案書・稟議書・報告書・請求書まで一気に出力するためのアプリです。本書は、導入手順・仕組み・必要なデータ・できることをまとめています。

対象アプリ：DPS試算アプリ（Streamlit版）／このアプリで扱う中核の計算式は変更されていません。

1

このシステムの概要

「何をするアプリか」を最初に押さえておくと、以降の入力がスムーズです。

本アプリは、顧客企業の電力検針票（過去12ヶ月）を起点に、空調のデマンド制御機器を導入した場合の 年間削減額・投資回収期間を試算します。数値を確定させると、そのまま 提案書（PowerPoint）・稟議シート（Excel）・削減実績レポート・請求書などを出力できます。

📥

入力

検針票12ヶ月＋顧客名・業態。あれば室外機リスト（型番）で精緻化。

🧮

試算

基本料金（デマンド／力率）＋電力量の削減を実データベースで算出。

📊

出力

提案書・稟議Excel・削減レポ・請求書・スライド貼付PNGを生成。

💾

管理

案件の保存・検索・比較、入力条件のスナップショット、バックアップ。

ポイント：入力できるデータが増えるほど「業界平均の仮置き」から「その現場の実データ」に切り替わり、算出がリアルになります。 まずは検針票だけでも試算でき、機材情報が無ければ概算モードで当たりを付けられます。
2

導入手順（初回セットアップ）

Windows PC を想定。フォルダを丸ごと受け取り、以下を1回だけ実施します。

Python を入れる（未導入の場合）

python.org から Python 3.9 以上をインストール。インストーラの「Add python to PATH」にチェック。

※ このPCでは python が旧版(2.7)のことがあるため、コマンドは必ず py -3 を使います。

必要ライブラリを入れる（初回のみ）

アプリのフォルダを開き、そこでコマンド プロンプトを起動して実行：

# アプリのフォルダ内で 1 回だけ
py -3 -m pip install -r requirements.txt

アプリを起動する

start.bat をダブルクリック。または、コマンドで：

py -3 -m streamlit run app.py

自動でブラウザが開きます。開かない場合は表示された http://localhost:8501 を開いてください。停止は Ctrl + C。

起動できない／不安定なとき：「Starting the CLR failed…」等が出ることがありますが一時的なものです。 ターミナルを開き直すか PCを再起動すると解消します。

同梱の はじめにお読みください.txt・操作マニュアル_本文.md も併せてご参照ください。

3

最短5ステップ（すぐ提案書を出す）

  1. 検針票を取込 … サイドバーの「検針票データ」でExcelをアップロード（無ければサンプル値で試せます）。
  2. 顧客名・業態を選ぶ … 会社名（御中）と業態を指定。業態で空調割合の標準値が自動反映。
  3. 台数を入れる … 室外機の総台数・制御台数。分からなければ「機材情報なし→検針票から概算する」をON。
  4. 初期費用 … 分かる範囲で入力（概算モードなら自動で概算）。
  5. 「試算実行」 … 回収年数などを確認し、PowerPoint/Excel をダウンロード。
迷ったら ②顧客名・業態 → ①検針票 → ③台数 → ⑤試算実行 だけでOK。あとから精緻化できます。
4

最低限インポートが必要な要素

これだけあれば試算できます。検針票の月次データ（12ヶ月分）＝以下の7列が必須です。

列名（必須）意味・使いどころ
月検針年月。2025/04 または 202504 形式。
契約電力(kW)基本料金の基準。デマンド削減後の契約見直し効果の起点。
最大需要電力(kW)デマンドのピーク。削減量の要（下の「リアルになる値」参照）。
使用量合計(kWh)月間の使用電力量。電力量料金の削減の基礎。
力率基本料金の力率調整（割引/割増）に使用。
基本料金単価(円/kW)契約電力×単価×力率調整＝基本料金。
電力量単価(円/kWh)使用量×単価＝電力量料金。

加えて、画面側で 顧客名 と 業態 を選べば試算実行できます。 取込用テンプレートはアプリ内の「インポート用テンプレDL」から入手できます。

検針年月の書式：2025年4月 / 2025/04 / 202504 のいずれでもOK。列名（見出し行）は変更しないでください。
5

この値を入れると"かなりリアル"に

必須項目は業界平均などの「仮置き」で動きますが、以下を入れるほどその現場の実データに切り替わり、算出精度が上がります。

入れると効く値何がリアルになるか区分
室外機リスト（型番付き）
機器ID/場所・メーカー・型番・制御可否(○/×)		 型番→定格（冷房能力）から容量加重で制御比率を精緻化。台数だけの概算より削減量が実態に近づきます。制御可否(○/×)で対象機器を指定。 精度↑
最大需要電力（実測） デマンド削減が実測ピークベースに。空欄だと使用量比から推定になるため、実測があるほど基本料金削減が正確に。 精度↑
力率（実測） 基本料金の力率割引/割増を実態に合わせて反映（0.85基準）。優良な力率の顧客ほど効果が精確に出ます。 精度↑
基本料金単価（月別） タリフの実値を月別に入れると、季節でタリフが変わる契約でも基本料金がリアルに。 精度↑
燃料費調整額（円/kWh・月別） 電力量単価へ自動加算して反映。マイナス月もそのまま反映され、実際の従量単価に近づきます。 精度↑
再エネ賦課金（円/kWh） 同じく電力量単価へ自動加算して反映。入れないと自動で除外されます。 精度↑
業態の空調割合（手動指定） 「その他※数値を指定する」を選ぶと、最大デマンド／使用量に占める空調割合を手入力でき、業態平均より実態に寄せられます。 精度↑
概算の前提（概算モード時）
標準室外機の定格出力(kW/台) など		 概算台数＝空調ピーク÷定格出力。定格出力・制御比率・単価を案件ごとに手入力して、概算でも現場感に合わせられます。 精度↑
任意だが現状は計算未反映の欄：「夏季ピーク／平日昼間／夜間休日」の時間帯別使用量・単価、および「市場価格調整項」は テンプレに欄はあるものの現時点では試算に反映されません（将来拡張用・空欄でOK）。上表の「精度↑」項目とは別扱いです。
まとめ：最短は「必須7列＋顧客名・業態」。より本気の提案には ①室外機リスト（型番）②実測の最大需要電力・力率 ③燃料費調整額・再エネ賦課金 を入れると、数字の説得力が大きく変わります。
6

これで何ができるか（出力物）

試算が確定すると、提案〜運用後まで一通りの資料をこのアプリから出力できます。

📑

提案書（PowerPoint）

KPIカード・分析手法・デマンドグラフ・10年収支・制御可否リスト等をフルビルド。

📗

稟議シート（Excel）

顧客提示用サマリー＋財務対応シート。稟議・社内検討用。

📉

削減実績レポート

運用後、前年比の削減実績を報告書（PDF/Excel）で作成。

🧾

請求書

成果報酬型の請求書（PDF/Excel）を作成。

🖼️

スライド貼付用PNG

グラフ・制御可否リスト（列選択可）・サマリカードを画像で書き出し。

🔎

概算3シナリオ

ベスト／標準／保守の3パターンで幅を提示（概算モード）。

🗂️

案件の保存・検索・比較

結果一覧に保存し、検索・2〜3件の横並び比較。

💾

スナップショット / バックアップ

入力条件を保存・呼び出し・受け渡し。全体データのバックアップ/復元。

出力ファイル名は エクスポート年月日_企業名_種別.拡張子（例：20260710_ゼンケン_提案書.pptx）で自動命名されます。

7

概算モード（機材情報が無いとき）

室外機や機材の情報がまだ無くても、検針票さえあれば概算できます。

  • 「機材情報なし→検針票から概算する」をONにすると、検針票の空調ピーク負荷 ÷ 標準室外機の定格出力から台数を推定し、初期費用・回収まで自動で概算します。
  • 「概算の前提（この案件用に調整）」で、定格出力・制御比率・材料費/工事費/ハードウェア費を案件ごとに手入力できます（既定はマスタ値）。
  • 結果は 3シナリオ（ベスト／標準／保守）で幅を提示。提案書には自動で「概算です」の注記と概算の説明ページが入ります。
概算はあくまで環境を想定した概算です。実機材・実見積が取得できたら、室外機リスト（型番）等を入れて精緻化してください。
8

困ったとき・注意点

よくある質問

  • 台数や機材費が分からない … 「機材情報なし→検針票から概算する」をON。標準単価で初期費用まで概算されます。
  • インポートで「列名が違う」と出る … アプリ内の「インポート用テンプレDL」に合わせてください。検針年月は 2025年4月/202504 形式。
  • 削減レポで「前年」が空になる … 前年（対象年）の月次データが未取込。追加インポートか企業データから両年を揃えてください。
  • 出力できるもの … 提案書(PPTX)／稟議シート(Excel)／削減実績報告書(PDF・Excel)／見積書・請求書(PDF・Excel)／スライド貼付用PNG。

運用上の注意

  • 設定・マスタ・保存案件・企業データ・スナップショットは dps_app_data.json に保存されます。定期的にバックアップ（各種設定＞データ管理）を。
  • 全体復元は完全上書きです。復元前に必ずバックアップを取得してください。
  • 設定を変更した後は、反映確認のため アプリ（Streamlit）を再起動すると確実です。
