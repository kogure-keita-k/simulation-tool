# DPS 試算アプリ ― 管理者・引き継ぎガイド

保守・運用担当者向け。マスタ更新／バックアップ／同期／バージョン方針などをまとめます。

---

## 1. 起動・実行環境
- Python 3.9以上（Windows）。実行は **`py -3`** を使う（※このPCの素の `python` は 2.7 のため使わない）。
- 起動方法：`start.bat` をダブルクリック、またはコマンドで
  ```
  py -3 -m streamlit run app.py
  ```
  （初回のみ `py -3 -m pip install -r requirements.txt`）
- 途中で「Starting the CLR failed…」等が出て起動できないことがある＝一時的なもの。**PC再起動 or ターミナル開き直し**で解消。停止は Ctrl+C。

## 2. ファイル構成（配布は"フォルダまるごと"）
- `app.py` … 本体（試算・各画面・提案書/Excel/PDF生成）
- `slide_image_export.py` … グラフ・リスト・アイコン等のPNG生成
- `scripts/build_workbook.py` … Excel出力の整形
- `icon_*.png`, `image_*.png`, `*.png` … 提案書・概算で使う画像（支給/生成）
- `requirements.txt`, `start.bat`, `起動.bat`, `はじめにお読みください.txt`
- `dps_app_data.json` … **設定・マスタ・企業データ・保存案件**（重要データ）
- ドキュメント：`操作マニュアル_本文.md`, `算出ロジック仕様書.md`, `CHANGELOG.md`, 本書

## 3. ★2フォルダの同期（重要）
本アプリは2箇所で管理されている：
- 本番：`...\aircon-demand-estimator\aircon-demand-estimator\`
- 作業：`...\aircon-demand-estimator (4)\aircon-demand-estimator\`

**`app.py` / `slide_image_export.py` / 追加画像 を変更したら、必ず両フォルダへコピー（同期）**すること。片方だけだと挙動が食い違う。

## 4. データのバックアップ・初期化
- 設定・マスタ・企業データ・保存案件はすべて `dps_app_data.json` に入る。
- **定期的にこのファイルをコピー保管**（バックアップ）。
- 他社・他PCへ"まっさら"で配布したいときは `dps_app_data.json` を削除してから渡す（初回起動で既定値から再生成）。

## 5. マスタ・単価の更新（各種設定 → 計算ロジック変更 ほか）
- 計算ロジック変更：容量削減率、換算係数、CO2/杉、力率、工事費目安単価、**概算マスタ**（標準室外機kW・機材費/台・工事費/台・HW固定/台数比例・概算制御比率）
- 機材単価マスタ：見積の機材費
- 業態マスタ：業態ごとの空調比率（ac_peak / ac_kwh）
- ※ 概算3シナリオの想定値は `gaisan_settings.scenarios`（現状は画面編集UIなし。必要なら追加可）
- 会社情報（TEL/FAX/住所）は `quote_settings.company`

## 6. アイコン・画像の差し替え
- 提案書・概算で使う画像は**同名PNGで上書き**すれば差し替わる（透過PNG推奨）。
  - 例：`icon_gaisan1/2/3.png`（概算説明）、`icon_kucho.png`（室外機）、`image_sort.png`（7手法グリッド）、`image_sort2.png`（分析3カード）、`image_schedule.png`（導入スケジュール）
- 変更後は両フォルダへ同期。

## 7. バージョン方針（APP_VERSION）
改修の都度、規模に応じて `app.py` 冒頭の `APP_VERSION` を更新（サイドバー上部に表示）。
- 大（X.0.0）：新しい画面の追加、算出方法の追加・削除
- 中（x.Y.0）：算出方法の修正、実装済み機能・システムの修正
- 小（x.y.Z）：見た目の修正
- 例外：出力資料(PPTX等)は「機能追加」以外はすべて軽微（パッチ）

## 8. 改修時の鉄則
- **削減・電気料金・回収の中核式（calc_simulation）は変えない**（詳細は『算出ロジック仕様書.md』）。
- 変更後は最低限、構文チェック（`py -3 -m py_compile app.py`）と、可能ならヘッドレスでの出力生成確認を行う。
- 変更内容は `CHANGELOG.md` に追記。
